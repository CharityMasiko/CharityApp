Project Overview

Good [morning/afternoon], everyone. Today I'm excited to present my project: My Budget, a comprehensive personal budget tracking web application.

This is a full-stack application that helps users manage their finances, track expenses, and learn about financial literacy through an integrated chatbot.

The application features:

  • Role-based access (Users & Admins)
  • Real-time budget tracking
  • Interactive analytics and reports
  • Financial education content
  • AI-powered financial assistant chatbot

Use Cases - For Users

For regular users, the application serves several key purposes:

1. Personal Budget Management

Users can track daily expenses across multiple categories like food, transport, rent, and utilities. They can record income from various sources and set monthly budget limits for each category. The system monitors spending against budgets in real-time.

2. Financial Analysis & Reporting

Users can view their spending patterns through interactive charts, analyze category-wise expense breakdowns, track monthly trends, and see how much they're saving.

3. Financial Education

Users can access educational content created by administrators and get instant answers from our financial chatbot about budgeting, saving strategies, investing basics, and debt management.

Use Cases - For Administrators

Database Design

I chose SQLite as the database for this project.

The database consists of 7 main tables:

  • users - User accounts & authentication
  • expenses - Track all expenses
  • income - Record income sources
  • budgets - Monthly budget limits
  • education - Financial content
  • notifications - System alerts
  • savings_goals - User objectives

Features: Foreign key constraints, indexes for performance, data validation, and automatic timestamps.

7 / 17

Why SQLite? - Part 1

1. Simplicity & Zero Configuration

SQLite is file-based - no separate database server needed. This makes deployment incredibly easy with a single database file that can be easily backed up or transferred. Perfect for development - quick setup without complex database server installation.

2. Lightweight & Embedded

Very small footprint, minimal resources - ideal for small to medium applications. Self-contained with all database functionality in a single library. Works out of the box with Node.js without external dependencies.

8 / 17

Why SQLite? - Part 2

3. Performance for Small-Medium Scale

Fast read operations, excellent performance for moderate concurrent users. ACID compliant ensuring data integrity. Efficient for single-user or small team applications - perfect for personal budget tracking.

4. Cost-Effective

Completely free and open-source with no licensing costs. No hosting costs - no separate database hosting service needed. Minimal maintenance with low administrative overhead.

5. Suitable for This Project

Personal budget tracking is typically single-user or small family use. Data volume for personal financial data is relatively small. Ideal for MVP and prototype stage.

Chatbot - Frontend Implementation

On the frontend, the chatbot is implemented as a modal-based chat interface.

  • Users click "Ask Financial Assistant" button in Education section
  • Floating chat window appears with conversation history
  • Clear distinction between user messages and bot responses with timestamps
  • Users can type questions and press Enter or click send
  • Loading indicators show while waiting for responses
  • Graceful error handling if network connection fails
11 / 17

Chatbot - Backend Implementation

On the backend, I created a RESTful API endpoint at /api/education/chatbot

The process:

  • Validates input - checks message isn't empty and doesn't exceed 500 characters
  • Uses pattern matching to analyze user input for financial keywords
  • Detects topics: budgeting, saving, investing, debt management, expenses
  • Generates contextual responses with relevant financial advice
  • Provides helpful default response if no specific topic detected

Chatbot - Integration Flow

1. User clicks "Ask Financial Assistant" button
↓
2. Chatbot modal opens
↓
3. User types question and sends
↓
4. Frontend makes POST request to API
↓
5. Backend processes message (pattern matching)
↓
6. Backend generates financial response
↓
7. Response displayed in chat interface

The whole process happens in real-time, providing instant feedback.

Chatbot - Technical Details

From a technical perspective, the chatbot uses keyword-based pattern matching:

  • Budgeting: Recognizes "budget" or "budgeting" → explains 50/30/20 rule
  • Saving: Detects "save" or "saving" → explains emergency funds and automation
  • Investing: Identifies "invest" → covers retirement accounts and diversification
  • Debt: Finds "debt" or "loan" → explains avalanche/snowball methods

The system handles errors gracefully, provides user-friendly messages, and includes timestamp tracking for all responses.
